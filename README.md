#### как оно работает? давай разберемся!
1. на самом смартфоне работает ollama serve, пока все по стандарту на http://localhost:11434
2. для распознавания голоса используется whisper.cpp - base
3. engine ждет, пока ему отправят веб-запрос (для этого используется Flask) по адресу http://127.0.0.1:9999/toggle, это post-запрос, есть состояния - IDLE RECORDING PROCESSING, при запуске http://127.0.0.1:9999/toggle (состояние в этот момент сменилось с IDLE на RECORDING) Flask получает запрос, запускается функция toggle(), вызывается encoding.start_recording(), включая микрофон, recording_start_time = time.time() засекает временной промежуток, затем повторное нажатие проверяется duration, если > 120 сек сброс в IDLE, если < 0.5 сек выход, иначе PROCESSING. затем если в момент генерации (PROCESSING - передача на генерацию) был отправлен http://127.0.0.1:9999/toggle, прерывает все вызовом inputOllama.abort_generation(). Сразу запускается новая запись: encoding.start_recording(). также существует POST http://127.0.0.1:9999/abort, мгновенно обрывает генерацию и очищает очередь озвучки. Не начинает запись (в отличие от toggle в PROCESSING). Состояние не меняет (старый поток сам завершится и сбросит на IDLE)

4. пока что используется модель ministral-3:3b(в планах сделать меню с выбором)
5. рекомендуется использовать приложение key mapper для переназначения клавиш на смартфоне(ну или сам придумывай способ отправлять post запросы)

итого для запуска тебе потребуется скачать whisper.cpp - base, далее стянуть репо - https://github.com/ggml-org/whisper.cpp, затем собрать whisper_bin (по умолчанию android не содержит инструментов сборки), затем установить ollama, благо в pkg она есть. поставить саму модель ministral-3:3b. не забыть изменить пути для подключения whisper.cpp

в развитии планируется создание тулов для вызова инструментов, дабы модель самостоятельно могла делать действия на смартфоне пользователя.
